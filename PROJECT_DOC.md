# Part 0 · 创意设想与 OpenClaw 硬件延伸

## 创意设想

**核心念头**：把「一人蹦迪 / 独奏」时的灯光，做成**和个人强相关**的体验——灯光自动追踪人、根据音乐情绪变色、甚至响应心情与歌声，形成「一人专属舞台」。

**输入**：摄像头（人位置、动作、表情）+ 麦克风/音频（节奏、情绪、歌声强度）。  
**输出**：可移动追踪灯球（Pan-Tilt 云台 + RGB 灯球）+ 音响。  
**交互**：Discord/Telegram 发指令（如「蹦迪模式 电子」），OpenClaw 自主运行、24/7 待命。

**设计原则**：完全本地运行（无云、无订阅）、低成本（约 100–150 美元）、两天 Hackathon 可完成 MVP，突出 **OpenClaw 在硬件与 IoT 方向的延伸**——把代理的「感知 → 记忆 → 自动化」落到真实硬件上。

## 痛点需求

| 痛点 | 描述 | 目标用户 |
|------|------|----------|
| **没有舞台感** | 一人唱歌/蹦迪时房间灯光死板，毫无氛围 | 独居青年、音乐爱好者 |
| **专业灯光贵** | 演唱会级追光系统上万元，个人难以承受 | 预算有限、DIY 倾向 |
| **隐私焦虑** | 智能家居方案上传视频到云端，不愿接受 | 注重隐私的用户 |
| **现有灯光不智能** | 市面 RGB 灯条只能手动/简单声控，不懂人、不懂歌 | 追求体验的极客 |
| **配置门槛高** | 专业 DMX 舞台灯需要专业知识，普通人难上手 | 非专业用户 |

## 价值假设

- **用户真的想要「一人舞台感」** → 通过 Hackathon 演示收集反馈，目标：80% 观众表示「想要一个」。
- **本地追踪够快、够稳** → MediaPipe 帧率 ≥15 FPS，追光延迟 <200ms。
- **节奏同步可感知** → 用户盲测（同步 vs 随机闪），目标 90% 能感知「灯在跟歌」。
- **隐私是真实需求** → 问卷/访谈中 60% 认为「不上传视频」很重要。

## OpenClaw 硬件延伸（Hackathon 主题体现）

ShiningBall 是 **OpenClaw 在硬件与 IoT 方向的具象落地**：

- **感知层**：摄像头 + 麦克风 → 人体位置、表情、音乐 BPM/能量，全部由 OpenClaw 调用的本地技能处理。
- **记忆层**：OpenClaw 持久记忆用户偏好（如上次喜欢紫色、偏好「摇滚」模式），下次自动偏好的灯光风格。
- **自动化层**：根据「人在画面 + 音乐开始」自动进入舞台模式；根据「动作变慢」自动切 chill 模式；通过 Discord/Telegram 指令切换模式，无需手动点按。

**与社区用例的对应**：家居自动化（灯光控制）、传感器监控（摄像头+麦克风）、多设备协作（灯球 + 音响 + 消息通道）、完全本地与可审计——与 OpenClaw 的「sensing → memory → automation」一致，且硬件成本控制在 100–150 美元、两天可交付，适合 Hackathon 展示。

# Part 1 · 产品需求文档（PRD）

## 我们要做什么

ShiningBall 是一颗「有灵魂的灯球」。

它能看见你、追随你、读懂你的音乐和情绪，然后用灯光回应你。你在房间里跳舞，它的光束会锁定你、跟着你移动；你放的歌节奏快，它就跟着闪爆；你跳累了坐下来，它会慢慢变暖、变柔。

**一句话**：用约 150 美元和一个周末，把万元演唱会追光灯体验带回家。

## 它是怎么工作的

```
你 ──── 摄像头看到你的位置 ────── 灯球转头追光打在你身上
     ├─ 麦克风听到音乐节奏 ────── 灯球跟着节拍变色闪烁
     └─ 摄像头读取你的表情 ────── 灯球随心情调整氛围
                  ↓
         所有计算都在本地完成
         OpenClaw 自主运行、无需云端
```

## 为什么要做这个

**用户是谁**：独居青年、音乐爱好者、DIY 极客，18-35 岁，住在出租屋或小工作室，追求生活仪式感，注重隐私。

**他们的痛**：

- **没有舞台感**。一个人唱歌蹦迪，房间灯光死气沉沉，完全没有氛围，这是最核心的痛点
- **专业灯光买不起**。演唱会级追光系统上万元，个人用户根本无法承受
- **隐私焦虑**。Alexa、Google Home 这些方案要上传视频数据到云端，很多人不愿意
- **现有灯光太傻**。市面 RGB 灯条只能手动调模式或简单声控，不可能真正"懂你"
- **配置门槛高**。专业 DMX 舞台灯光需要专业知识，普通人根本学不会

**需求优先级**：

- ✅ **一定要有**：灯球追踪人体位置（云台自动转）、音乐节奏同步灯光、完全本地运行
- 🟡 **最好有**：表情情绪检测影响灯色、Discord/Telegram 指令控制
- ⚪ **可以没有**：歌声强度检测、OpenClaw 记住用户偏好色

## 凭什么是我们

核心价值：**把万元体验做到千元以内，而且完全属于你自己**。

对比市面产品：

- **Philips Hue Sync**（¥3000+）：能同步音乐，但不能追踪人，需要云账户
- **Nanoleaf**（¥2000+）：好看，但不能追踪人，不能根据表情变色，封闭生态
- **LIFX**（¥1500+）：同上，API 有限

**ShiningBall 全部做到**：追踪人 ✓ 音乐同步 ✓ 表情响应 ✓ 100% 本地 ✓ 全开源可定制 ✓ —— 总共 ¥550。

## 怎么验证价值

- **用户想要"一人舞台感"吗？** → 先走创造需求的路线，基于脑洞开始思考。
- **本地追踪够快吗？** → 测 MediaPipe 帧率，目标 ≥15 FPS、追踪延迟 <200ms
- **节奏同步效果好吗？** → 让用户盲测（同步 vs 随机闪），目标 90% 能感知到"灯在跟歌"
- **隐私是真需求吗？** → 问卷比对，目标 60% 认为"不上传视频"很重要

# Part 1.5 · 产品形态与工业设计（智能灯球 / 一体机）

> 参考 **Apple 家居**（HomePod / HomePod mini）与 **小米生态**（智能音箱、带屏音箱），把「灯球」升级为一台有智能感、可摆上桌面的**一体式设备**：小屏 + 音响 + 多效果灯光 + **摄像头** + **灵活转动**。整体**好看、带科技感与屏幕感**，灯光样式多、可打在场景/人/锚点位置。

## 设计定位

**一句话**：一台「会追光、会跟节奏、会看心情」的桌面智能灯 + 音箱 + 小屏助手，外观接近高端智能音箱，能力是私人舞台灯光中枢；**有摄像头、可灵活转动，灯光能打在人和锚点上**。

| 维度 | 方向 |
|------|------|
| **形态** | 矮圆柱或圆角柱，织物/磨砂机身，便于融入书架/桌面，像 HomePod、小米 Sound 那样「摆着就好看」 |
| **一体** | 音响 + 灯光 + 小屏 + **摄像头**集成在一个机身内，减少外接线缆和零碎感 |
| **智能感** | 小屏有**屏幕感**，显示状态/模式/波形；灯光**样式多**（呼吸、节奏、追光、打在人身上/锚点）；支持指令与（可选）语音 |
| **科技感** | **摄像头**（正面或顶缘）用于追踪与场景感知；**灵活转动**（Pan-Tilt 云台），灯光可定向打在场景/人/锚点位置 |

## 参考产品气质

| 参考 | 可借鉴点 |
|------|----------|
| **Apple HomePod / HomePod mini** | 织物网面、圆润矮柱、顶部触控区与 LED 反馈、一体成型、哑光底座 |
| **小米 Sound / 小米智能音箱 Pro** | 金属网罩或织物、顶部环形灯带、紧凑体积、小屏或灯环显示状态 |
| **带屏智能音箱** | 小屏做状态与可视化（波形、模式名），**屏幕感**强、不喧宾夺主 |

## 推荐形态：矮圆柱 + 顶环灯 + 小屏 + 摄像头

**整体**：矮圆柱体，直径约 12–14 cm、高约 15–18 cm，可放在 **Pan-Tilt 云台上灵活转动**实现追光，灯光可从顶环打出、**打在场景、人身上或锚点位置**。

| 区域 | 设计要点 |
|------|----------|
| **顶部** | **环形 RGB 灯带**（360° 可见）：呼吸、节奏闪、追光时配合机身转向；环心为**小圆屏**（约 2–3 英寸），有**屏幕感**，显示模式/波形/BPM；灯光可**定向/聚光**打在人或锚点 |
| **侧面** | **织物网面**或**透光磨砂**：透声 + 机身可泛光；颜色以灰白/深灰为主，易融入家居 |
| **摄像头** | 正面或顶部环缘处**小型镜头**，用于人体追踪与场景感知，外观融入、不突兀 |
| **底座** | 哑光塑料或金属，内藏 Pan-Tilt 舵机时**灵活转动**；或设备固定、仅顶部「灯环+屏」区域做小幅俯仰 |
| **正面/侧面** | 可选：隐藏式麦克风孔、一颗状态 LED（待机/聆听/执行） |

## 灯光效果定义（样式多、可打在场景/人/锚点）

| 效果名 | 表现 | 典型场景 |
|--------|------|----------|
| **呼吸** | 单色或渐变缓慢明暗，顶环 + 可选机身泛光 | 待机、 chill、演奏柔板 |
| **节奏闪** | 顶环随 BPM 闪烁，颜色随风格（电子=紫红，民谣=暖橙） | 蹦迪、放歌 |
| **追光** | 机身**灵活转动**对准人 + 顶环高亮或**聚光打在人身上** | 跳舞、独奏 |
| **锚点光** | 灯光打在**具体锚点位置**（如桌面、墙面某点），多灯光样式 | 氛围、展示 |
| **氛围** | 全向柔光、色温随时段或情绪（晨=中性白，晚=暖黄） | 日常、WFH |
| **状态** | 小屏旁或环上简单指示（如蓝=听令，绿=执行，琥珀=待机） | 全场景 |

## 音响与屏

| 模块 | 说明 |
|------|------|
| **音响** | 内置全频单元，蓝牙输入 + 本地播放；音质「够用」、人声清晰；与灯光节奏同步时由同一设备出声+出光，一体感强 |
| **小屏** | 圆形或圆角矩形，显示：模式图标、BPM/节拍可视化、简单波形或「正在听」；支持触控切换模式，与 Discord/Telegram 并行 |

## 材质与尺寸（目标）

| 项目 | 建议 |
|------|------|
| **材质** | 侧面织物或透光磨砂塑料，顶环亚克力导光，底座哑光塑料/金属；整体好看、带科技感与屏幕感 |
| **尺寸** | 直径 ≤14 cm，高 ≤18 cm，便于桌面/书架摆放与云台承载 |
| **重量** | 尽量 ≤600 g，若含云台则整体考虑稳定性 |

## 与现有 Hackathon 方案的衔接

- **MVP 阶段**：可继续用「树莓派 + 云台 + WS2812B 灯环 + 外接音箱 + 摄像头」实现逻辑，在灯环外**加一层外壳**（3D 打印矮圆柱 + 织物/磨砂罩）和**小屏**、**摄像头**，逼近本设计。
- **概念/演示视频**：按本节的「矮圆柱 + 顶环灯 + 小屏 + 摄像头 + 灵活转动、灯光打在人/锚点」在 `prompts/01_产品图_风格锚点.md` 中生成产品图与极梦视频，统一为**好看、科技感与屏幕感**的一体机形象。

# Part 2 · 场景刻画（效果与应用场景）

## 效果应用场景概览

| 场景 | 谁在用 | 核心效果 | 关键指标 |
|------|--------|----------|----------|
| 一人蹦迪 | 独居青年、派对爱好者 | 追光打在身上 + 节奏闪爆 + 累了自动变柔 | 追光 <200ms，节拍同步误差 <50ms |
| 独奏/演唱 | 乐手、主播、K 歌用户 | 灯光始终照人 + 随段落/音量变色 | 坐姿稳定 <5° 抖动，过渡平滑 |
| 日常氛围 | WFH、夜猫子 | 人在即亮、人走即暗，按时段/活动自适应 | 状态切换准确率 >90%，误触发少 |

下面为每个场景做具体刻画。

## 场景一：一人蹦迪 ✦ 核心场景

> 周五晚上 9 点。小明，25 岁程序员，独居，下班回到出租屋。

他打开 Discord，输入 **"@ShiningBall 蹦迪模式 电子"**。

灯球亮了。先是柔和的呼吸灯，等待信号。他打开蓝牙音响放上电子乐——128 BPM 的 bass drop 砸下来的瞬间，灯球炸开紫红色闪烁，和节拍完美同步。

他走到房间中央。灯球的云台"扭头"对准他，一束光稳稳打在身上。他开始跳，灯球跟着他左右移动。音乐进入高潮段——灯光闪烁加速，亮度拉满，整个房间变成他的私人 club。

20 分钟后他跳累了，动作明显放慢。系统检测到运动幅度下降，灯光自动从紫红切换到暖黄巨口慢呼吸，音乐也跟着切到 lo-fi。

**他的感受**："这就是我的私人 club。"

**关键体验指标**：追光响应 <200ms，节拍同步误差 <50ms。

## 场景二：独奏练习

> 周末下午。小红，28 岁，业余吉他手。

她在 Telegram 发了一条 **"演奏模式 民谣"**。灯球切到暖色调，缓慢脉动。

她坐在椅子上弹吉他。灯球精准锁定她，光束始终打在她身上。弹到激情段落——音量上去了、频率变高了——灯球跟着变亮，色温偏向冷白。指弹轻柔段落又回到暖橙色。

她拿手机录了一段视频。因为有动态灯光变化，视频观感直接上了一个档次——"感觉像在录音棚。"

**关键体验指标**：追光稳定（坐姿抖动 <5°），灯光过渡平滑无突变。

## 场景三：日常氛围灯

> 工作日。小李，30 岁，WFH 远程工作者。

不需要发任何指令。早上 9 点他坐到桌前，系统检测到人，自动切换中性白光柔和照明。下午 6 点他站起来活动，灯光逐渐变暖。晚上放音乐，自动进入音乐同步模式。深夜人不在画面超过 10 分钟，灯球自动灭了。

**全天零操作，灯光自己懂。**

**关键体验指标**：状态切换准确率 >90%，一天误触发 <1 次。

## 情绪与灯光的关系

| 你在做什么 | 灯球怎么反应 | 颜色 |
|---|---|---|
| 快歌蹦迪（BPM >120） | 高频闪烁，亮度拉满 | 紫 · 红 · 电蓝 |
| 慢歌放松（BPM <90） | 缓慢渐变呼吸 | 暖橙 · 翠绿 · 深蓝 |
| 笑了 / 很兴奋 | 亮度上升，偏暖 | 金黄 · 亮紫 |
| 安静专注 | 亮度降低，偏冷 | 冰蓝 · 中性白 |
| 大幅度舞蹈 | 追光加速，饱和度拉高 | 彩虹渐变 |
| 累了、慢下来 | 追光减速，暖色过渡 | 琥珀 · 奶黄 |
| 飙高音 | 爆闪！ | 白色闪光弹 |
| 安静间奏 | 暗淡微光 | 暗紫 · 深海蓝 |

## 我们交什么

## 怎么算做完了

**必须通过的验收项（功能层面）**：

1. **灯球追人** — 人在 3 米范围内移动，灯球持续对准人体中心，偏差 <15°
2. **音乐同步** — 播放 3 首不同 BPM 的歌，灯光闪烁频率与节拍匹配率 >80%
3. **灯光变色** — 快歌（>120 BPM）出红紫系，慢歌（<90 BPM）出蓝绿系
4. **指令控制** — Discord 发 "蹦迪模式"，5 秒内灯球启动响应
5. **完全离线** — 断掉 WiFi，除了 Discord 通道以外所有功能正常

## 系统架构

整个系统分四层：

**第一层 · 用户交互**
Discord Bot / Telegram Bot / 命令行 — 你发指令的地方。

**第二层 · OpenClaw 核心**（Node.js）
接受指令、管理技能插件、存储用户偏好记忆、路由消息。

**第三层 · ShiningBall 引擎**（Python）
真正干活的地方，四个模块：

- `vision.py` — 摄像头 + MediaPipe → 人体位置坐标 + 面部表情分数
- `audio.py` — 麦克风/音频输入 + librosa → BPM、能量、频谱、是否在拍点上
- `fusion.py` — 把视觉和音频信号融合 → 计算出灯该怎么变（颜色、亮度、闪烁、云台角度）
- `hardware.py` — 把计算结果翻译成电信号 → 控制舵机转向 + LED 亮灯

**第四层 · 硬件**
Pan-Tilt 云台（2 个舵机）+ WS2812B RGB LED + Pi Camera + USB 麦克风。

## 两天 Hackathon 时间表

### Day 1 — 让它动起来（8 小时）

**第 1 小时 · 硬件组装**
把 Pan-Tilt 云台装好，灯球固定在上面，摄像头装在旁边。接好杜邦线。
> ⚠ 风险：舵机卡壳 → 检查是否给了独立 5V 供电

**第 2 小时 · 环境搭建**
Pi 烧系统、装 Python 环境、装 OpenClaw（`pnpm install`）。
> ⚠ 风险：pip 安装慢 → 提前离线下载 wheel 包

**第 3-4 小时 · 视觉追踪**
写 `vision.py`：打开摄像头 → MediaPipe Pose 检测 → 输出人体中心归一化坐标 (x, y)。

核心逻辑：用鼻子点 + 双肩中心作为追踪基准点。模型选 `model_complexity=0`（轻量版，Pi 跑得动）。分辨率降到 320×240 保帧率。
> ⚠ 风险：Pi 上 MediaPipe 慢 → 降分辨率到 240×180

**第 5 小时 · 云台追光**
写 `hardware.py` 中的云台部分：拿到人体坐标 → 映射成 Pan（左右）和 Tilt（上下）角度 → 输出 PWM 信号驱动舵机。

关键细节：坐标到角度的映射需要标定。加低通滤波避免舵机抖动。设死区（角度变化 <3° 不动）。
> ⚠ 风险：舵机抖 → 加 100μF 电容 + 软件滤波

**第 6-7 小时 · 音频分析**
写 `audio.py`：USB 麦克风实时采集 → librosa 检测 BPM 和能量 → 输出节奏特征。

核心指标：BPM（决定闪烁速度）、RMS 能量（决定亮度）、频谱重心（决定色温冷暖）、是否在拍点上（决定是否闪一下）。
> ⚠ 风险：延迟大 → FFT 窗口从 1024 缩小到 512

**第 8 小时 · 基础联调**
把追踪和音频分别跑通：人移动灯球追、放歌灯光闪。今天两个功能可以独立工作。
> ⚠ 风险：多线程冲突 → 用 asyncio 或独立进程隔离

### Day 2 — 让它聪明起来（8 小时）

**第 1-2 小时 · 信号融合**
写 `fusion.py`：把追踪坐标、音频节奏、表情分数合到一起，统一输出灯光参数。

优先级：追踪（最高，1.0）> 节拍（0.8）> 音量（0.6）> 情绪（0.4）。信号冲突时高优先级覆盖低优先级。

**第 3 小时 · 预设模式**
写 `modes.py`：定义四个模式——蹦迪（高饱和快闪）、演奏（暖色柔和）、氛围（自适应白光）、chill（暖黄慢呼吸）。

**第 4 小时 · OpenClaw 集成**
写技能插件：Discord 收到 "蹦迪模式" → OpenClaw 调用引擎 → 切换模式。用简单 HTTP 桥接 Node.js 和 Python。

**第 5 小时 · 疲劳检测**
在 `fusion.py` 加逻辑：连续 30 帧运动幅度低于基线 50% → 判定"累了" → 自动切 chill 模式。

**第 6-7 小时 · 体验调优**（最重要！）
灯光颜色参数打磨、过渡动画平滑、追光速度微调、测试不同风格音乐的表现。这两个小时决定最终效果。

**第 8 小时 · 打包演示**
录 demo 视频、写 README、清理代码、准备现场演示。

## 接线方式

```
Raspberry Pi GPIO 引脚分配：

    GPIO 12 (PWM0) ──→ Pan 舵机信号线（橙色）
    GPIO 13 (PWM1) ──→ Tilt 舵机信号线（橙色）
    GPIO 18         ──→ WS2812B LED 数据线（绿色）
    5V              ──→ 舵机供电（红）+ LED 供电（红）
    GND             ──→ 舵机地线（棕）+ LED 地线（白）

注意事项：
  · LED 超过 30 颗时，必须用独立 5V 电源（2A 以上），不能从 Pi 取电
  · GPIO 18 到 LED 数据线之间串一个 330Ω 电阻保护
  · 舵机供电端并一个 100μF 电容防抖
```
